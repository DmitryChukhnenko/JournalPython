# University Schedule System

Консольная система управления расписанием учебных занятий.

## 📋 Описание

Система позволяет управлять:
- Учебными группами
- Предметами (дисциплинами)
- Аудиториями
- Преподавателями
- Расписанием занятий

## 🏗️ Структура проекта

```
university_schedule/
```

## 🚀 Запуск

### Требования
- Python 3.8+
- Только стандартная библиотека (stdlib)

### Установка

1. Клонируйте репозиторий или скопируйте файлы проекта
2. Перейдите в директорию проекта:
   ```bash
   cd university_schedule
   ```

### Запуск приложения

```bash
python main.py
```

## 📖 Использование

После запуска вы увидите главное меню:

```
============================================================
              📚 Система управления расписанием
============================================================

[1] 📋 Группы
[2] 📖 Предметы
[3] 🏫 Аудитории
[4] 👨‍🏫 Преподаватели
[5] 📅 Расписание
[0] 🚪 Выход

Выберите действие:
```

### Пример рабочего процесса

1. **Добавьте группу:**
   - Выберите `[1] Группы`
   - Выберите `[1] Добавить группу`
   - Введите название, количество студентов, куратора

2. **Добавьте предмет:**
   - Вернитесь в главное меню
   - Выберите `[2] Предметы`
   - Добавьте новый предмет

3. **Добавьте преподавателя:**
   - Выберите `[4] Преподаватели`
   - Добавьте нового преподавателя

4. **Добавьте аудиторию:**
   - Выберите `[3] Аудитории`
   - Добавьте новую аудиторию

5. **Создайте занятие:**
   - Выберите `[5] Расписание`
   - Выберите `[1] Добавить занятие`
   - Следуйте инструкциям для выбора группы, предмета, преподавателя и аудитории

6. **Проверьте конфликты:**
   - В разделе "Расписание" выберите `[4] Проверить конфликты`

## 💾 Хранение данных

Все данные автоматически сохраняются в файл `data/schedule.json` при:
- Добавлении/удалении сущностей
- Выходе из приложения (опция `[0]`)

Файл имеет следующую структуру:
```json
{
  "teachers": [...],
  "groups": [...],
  "subjects": [...],
  "classrooms": [...],
  "schedule": [...]
}
```

## 🔧 Архитектура

Проект следует принципам чистой архитектуры:

- **Models** (`models/`): Бизнес-сущности с валидацией
- **Repository** (`repository/`): Абстракция доступа к данным
- **Core** (`core/`): Бизнес-логика и менеджеры
- **UI** (`ui/`): Пользовательский интерфейс

### Ключевые принципы

1. **Разделение ответственности**: UI не содержит бизнес-логики
2. **Инкапсуляция**: Приватные атрибуты и публичные методы
3. **Валидация**: Все данные проверяются при создании и изменении
4. **Чистые функции**: Функции валидации не имеют побочных эффектов

## ⚠️ Обработка ошибок

Система обрабатывает следующие ошибки:
- Некорректный ввод пользователя
- Конфликты расписания (пересечение по времени)
- Отсутствие необходимых сущностей
- Проблемы с файловой системой
